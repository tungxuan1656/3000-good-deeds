# Documentation Maintenance Checklist

Use this checklist during release prep, major refactors, and monthly maintenance.

## Keep

- Keep documents that are actively used during coding, review, onboarding, or deployment.
- Keep only one source of truth per topic.

## Remove or Archive

- Remove documents that duplicate existing standards.
- Archive historical reports that are no longer used operationally.
- Remove docs that no longer match runtime behavior.

## Verify Completeness

- Product intent and feature behavior are documented.
- API contracts are up to date with runtime routes.
- DB schema docs reflect current migrations.
- Frontend and backend standards are current.
- Release/deploy checklist is current.

## Verify Quality

- Naming follows project conventions (`kebab-case`, stable paths).
- Docs are easy to discover from README indexes.
- Internal links are valid.
- Terminology is consistent across frontend and backend docs.

## Required Outputs per Maintenance Round

- List of docs removed/merged.
- List of docs updated.
- Remaining known gaps and priorities.
