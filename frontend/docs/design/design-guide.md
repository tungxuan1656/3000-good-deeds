# Design Guide

## Objective

Define consistent visual and interaction rules for the frontend.

## Core Principles

- calm and readable interfaces
- privacy-first and reflection-friendly tone
- accessible contrast and spacing
- mobile-first layout decisions

## Visual Guidelines

- use semantic color tokens from the design system
- keep typography hierarchy consistent
- use iconography consistently across features
- avoid noisy visual effects in primary journaling flows

## Interaction Guidelines

- make primary actions clear and low-friction
- use confirmation for destructive actions
- show clear empty/loading/error states
- prefer predictable UI behavior over novelty
