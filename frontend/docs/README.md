# Frontend Docs

Frontend documentation is split into three groups:

- `frontend/docs/standards/`: coding patterns, conventions, and review rules.
- `frontend/docs/product/`: frontend feature business specs.
- `frontend/docs/design/`: UI/UX and design references.

## Suggested Reading Order

1. `frontend/docs/standards/project-folder-structure.md`
2. `frontend/docs/standards/naming-and-conventions-pattern.md`
3. `frontend/docs/standards/code-review-guide.md`
4. `frontend/docs/product/` (feature-specific)
5. `frontend/docs/design/design-guide.md`
